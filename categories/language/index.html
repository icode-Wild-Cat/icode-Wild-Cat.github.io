1.3 class name

Python naming convention for classes are the same as any other programming language like C#, Java and C++,etc.

  1. Need to follow the camelCase convention
  2. End with “Error” when writing any classes for an exception
  3. Can give a class name like a function to call the class from somewhere or callable
  4. Bulitin class be lowercase

1.4 file name

  1. Should choose a short name as file name
  2. All lowercase
  3. Can also use underscore ( _ ) with the file name

1.5 method name

  1. All lowercase
  2. Separate by an underscore( _ ) when there are multiple words in this method name
  3. Non-public method name start with an underscore( _ )
  4. Start with two undercase( _ ) when wanna alter a method mame

1.6 constant name

  1. All uppercase
  2. Separated by an underscore( _ ) when there are multiple words in this constant name

1.7 package name

  1. All lowercase
  2. Separated by an underscore( _ ) when there are multiple words in this package name
  3. best to use one word as package name

1.8 object name

  1. All lowercase
  2. choose a very stort name
  3. Separated by an underscore( _ ) when there are multiple words in this object name

1.9 module name

  1. All lowercase
  2. Separated by an underscore( _ ) when there are multiple words in this module name
  3. best to use one word as module

1.10Global variable name

  1. All lowercase
  2. Separated by an underscore( _ ) when there are multiple words in this globle variable name

Snake_case naming

Snake_case naming is Separated by an underscore( _ ) when there are multiple words in this module name

1.11 Arguments of method

  1. Use argument named ‘self’ as their first argument in case of Instance methods
  2. For class methods, use ‘cls’ as their first argument

General naming conventions

  1. Always give a meaning full name for any of the python objects. Don’t give a name like x,y, or z, etc
  2. Don’t give space in between object name instead of using underscore(_) when there are more than one-word presents.
  3. Better to use camelCase only when it makes sense or else try avoiding more use of camelCase.

We commonly need to execute calculations on numeric data types, including complex numbers, whether working on machine learning, or scientific computations.How to write a Python Program to create a complex number

1. Complex Numbers in py

  • When both the real(实部) and imaginary(虚部) parts are given, the complex() method produces a complex number; otherwise, it converts(转换) a string to a complex number.
  • Any integer that can be expressed as (a+b j), where a and b are real numbers, is referred to as a complex number. In this case, the imaginary number j is defined as being equal to the square root（平方根） of -1.
  • The square roots of negative numbers（负数）are frequently utilized in calculations involving complex numbers, which typically occur in pairs（复数通常成对出现）.
  • By simply declaring（声明） a variable and assigning an expression of the type (a+bj), we can define complex numbers in Python. Here, “a” and “b” should be literal（文字） numbers, while “j” can be any letter of the alphabet.
